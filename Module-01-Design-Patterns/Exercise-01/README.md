# Exercise 1: Implementing the Singleton Pattern

## Objective
Ensure that a logging utility class in the application has only one instance throughout the application lifecycle to ensure consistent logging.

## Technologies Used
- Java 17

## Files Included
- `Logger.java`: The singleton class with a private constructor and a static instance.
- `TestSingleton.java`: The test class to verify that only one instance of `Logger` is created and used across the application.

## Output Screenshot
